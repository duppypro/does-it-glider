Serve Utility (serve.ts)

Domain: Local Development & Process Management

Overview

The Serve utility provides a secure, HTTPS-enabled local development server with real-time TUI monitoring. It relies strictly on stateless OS-level data (ps aux) rather than conversational history.

Bounded Context & Rules

  • Stateless Tracking: Servers are identified dynamically by scanning the host OS. A server belongs to the active context if its served directory resides within process.cwd().
  • Security: Scans target directories for .env files and prompts for confirmation before exposing them over the network.
  • Deduplication: Prevents multiple HTTP servers from spawning on the exact same absolute directory path.

Architectural Layers

Application: Command parsing (/serve --hide, --kill).

Infrastructure: Sub-process spawning, lsof PID lookups, and session log visibility polling.

Presentation: ANSI-safe string padding and TUI widget rendering.
